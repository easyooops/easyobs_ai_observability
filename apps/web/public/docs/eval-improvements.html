Improvement Packs — Detailed Manual

Improvement Packs — Full Feature Manual

Improvement Packs translate evaluation failures into actionable engineering tasks. Each pack identifies the root cause of a quality issue and provides categorized, effort-graded suggestions for fixing it.

Image: Improvements page — grouped cards with effort badges and workflow states

How Improvement Packs Work

  1. An evaluation run produces FAIL or WARN verdicts with reasoning
  2. The system analyzes the reasoning and maps it to a cause code
  3. Each cause code maps to a primary category and secondary detail
  4. Categories belong to one of 11 groups
  5. Each suggestion carries an effort level (Low / Medium / High)
  6. Packs are presented as actionable cards the team can track through workflow states

Effort Levels

LevelBadgeTypical TimeExamples
LowGreen< 1 hourAdjust prompt wording, add guardrail rule, tweak temperature
MediumYellow1 hour – 1 dayRestructure prompt template, add retrieval step, improve chunking
HighRed> 1 dayRedesign agent architecture, fine-tune model, add new data pipeline

Workflow States

Each improvement pack flows through these states:

StateMeaningAction
openNewly identified, awaiting triageReview and decide priority
in_progressTeam is actively working on the fixImplement the suggestion
resolvedFix applied, awaiting verificationRe-run evaluation to confirm
dismissedNot applicable or won't fixArchived (still visible in history)

Cause Code Mapping

The system uses 52 internal cause codes (generated by LLM judge reasoning analysis) which map to the 46 categories above. Multiple cause codes can map to the same category. The mapping ensures that even nuanced failure reasons are funneled into actionable categories.

UI Layout

Grouped Card View

  • Improvements are displayed as cards grouped by the 11 category groups
  • Each group header shows the group name and count of open items
  • Cards show: category name, effort badge, cause summary, suggested action
  • Click a card to expand full details and change workflow state

Filters

  • Effort: Filter by Low / Medium / High
  • State: Open / In Progress / Resolved / Dismissed
  • Group: Filter by specific category group
  • Run: Filter by source evaluation run

Integration with Runs

After resolving an improvement, best practice is to:

  1. Apply the fix to your agent code
  2. Re-run evaluation on the same traces
  3. Verify the FAIL/WARN verdicts have converted to PASS
  4. Mark the improvement as "resolved" once verified
