LangChain 통합

내장 EasyObsCallbackHandler를 사용해 LangChain 애플리케이션을 별도 설정 없이 트레이싱할 수 있습니다.

설치

pip install -e ".[langchain]"

이 명령은 다음을 함께 설치합니다: opentelemetry-api, opentelemetry-sdk, opentelemetry-exporter-otlp-proto-http, langchain-core.

옵션 A: 자동 모드 (권장)

init()에 auto=True만 지정하면, 명시적인 콜백 연결 없이도 LangChain의 모든 호출이 전역적으로 캡처됩니다.

from easyobs_agent import init

init(
    "http://localhost:8787",
    token="eobs_...",
    service="my-langchain-app",
    auto=True,              # auto_langchain + capture_io 동시 활성화
    session_id="sess_123",  # 선택: 기본 세션
    user_id="alice",        # 선택: 기본 사용자
)

# 끝입니다! 모든 llm.invoke(), chain.invoke(), retriever.invoke()가
# 코드 변경 없이 자동으로 트레이싱됩니다.
동작 원리: auto_langchain=True는 langchain_core.tracers.context.register_configure_hook을 통해 프로세스 전역 콜백을 등록합니다. LangSmith가 사용하는 것과 동일한 메커니즘입니다.

옵션 B: 명시적 콜백

좀 더 세밀하게 제어하려면 핸들러를 직접 전달합니다.

from easyobs_agent import init
from easyobs_agent.callbacks import EasyObsCallbackHandler

init("http://localhost:8787", token="eobs_...", service="my-app")

handler = EasyObsCallbackHandler(session_id="sess_123", user_id="alice")

# 개별 호출에 콜백 전달
result = llm.invoke("Hello", config={"callbacks": [handler]})
result = chain.invoke({"query": "..."}, config={"callbacks": [handler]})

캡처되는 데이터

LangChain 컴포넌트스팬 종류캡처 데이터
LLM / ChatModelllm모델, 프롬프트, 응답, 토큰, 비용
Chain / RunnableSequencechain체인 이름, 입력/출력
Retrieverretrieve질의, 검색된 문서

전체 예제

from easyobs_agent import init
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

# 한 번만 설정
init("http://localhost:8787", token="eobs_...", service="rag-bot", auto=True)

# 평소처럼 체인 구성 -- 트레이싱은 자동으로 동작
llm = ChatOpenAI(model="gpt-4o-mini")
prompt = ChatPromptTemplate.from_template("Answer: {question}")
chain = prompt | llm | StrOutputParser()

# 모든 invoke 호출이 자동으로 트레이싱됩니다
answer = chain.invoke({"question": "What is EasyObs?"})
