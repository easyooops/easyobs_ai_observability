인터랙션

인터랙션 페이지는 세션 단위 관찰과 사용자 단위 관찰을 하나의 탭 화면에 통합한 뷰입니다. 대화가 여러 턴에 걸쳐 어떻게 이어지는지(Sessions 탭)와, 개별 사용자가 LLM 에이전트와 어떻게 상호작용하는지(Users 탭)를 함께 살펴볼 수 있습니다.

개요

인터랙션은 에이전트와의 상호작용을 두 가지 관점으로 통합해 보여줍니다.

탭그룹 기준속성활용 예시
Sessions대화 / 세션 IDo.sess멀티턴 대화 분석, 턴 단위 리플레이
Users사용자 IDo.user사용자별 활동 요약, 비용 추적, 참여 패턴 분석

Sessions 탭

세션은 동일한 o.sess 속성을 공유하는 트레이스들을 묶은 단위로, 보통 여러 턴에 걸친 하나의 대화에 해당합니다. 계층 구조는 session > traces > spans입니다.

세션은 어떻게 만들어지나요?

같은 세션 ID를 공유하는 트레이스가 들어오면 세션이 자동으로 형성됩니다. 에이전트 코드에서 세션 속성만 설정하면 됩니다.

Python SDK

from easyobs_agent import init, traced, span_tag, SpanTag

init("http://127.0.0.1:8787", token="eobs_...", service="my-agent")

@traced("agent.turn")
def handle_turn(user_msg: str):
    span_tag(SpanTag.SESSION, "conv-abc-123")
    span_tag(SpanTag.USER, "user-456")
    ...

TypeScript / JS (OTLP)

span.setAttribute("o.sess", "conv-abc-123");
span.setAttribute("o.user", "user-456");

LangChain Callback

from easyobs_agent import init
init(session="conv-abc-123", user="user-456", auto_langchain=True)

세션 인스펙터 (Drawer)

세션 행을 클릭하면 인스펙터 패널이 열리며 다음을 보여줍니다.

  • 세션 메타데이터 (ID, 서비스, 사용자, 트레이스 수)
  • 모든 턴에 걸친 토큰/비용 합계
  • 에러 발생 수와 사용된 모델
  • 턴 시간 순 목록과 개별 트레이스 상세 페이지로의 링크

Users 탭

Users 탭은 사용자 식별자별로 활동을 집계합니다. o.user 속성이 부여된 모든 트레이스가 해당 사용자의 활동에 합산됩니다.

사용자는 어떻게 추적되나요?

사용자는 o.user 스팬 속성으로 식별됩니다. 세션과 함께 다음처럼 설정하세요.

# Python
span_tag(SpanTag.USER, "user-456")

# TypeScript / JS
span.setAttribute("o.user", "user-456");

Users 목록 컬럼

컬럼설명

Service가장 최근 서비스 이름

사용자 인스펙터 (Drawer)

사용자 행을 클릭하면 인스펙터 패널이 열리며 다음을 보여줍니다.

  • 사용자 식별자, 서비스, 세션/트레이스 수
  • 토큰/비용 집계
  • 에러 수와 사용 모델
  • 최근 트레이스 목록 (최신순, 최대 30건)

필터 & 검색

  • 시간 윈도우: 두 탭 모두 상단의 전역 시간 윈도우를 따릅니다.
  • 전역 검색: 세션 ID / 사용자 ID / 서비스를 부분 일치로 필터링합니다.
  • 좌측 사이드바 필터: 탭별 정확 일치 필터(Session ID 또는 User ID)

필요한 속성

속성용도필요한 탭/기능예시
o.sess세션 IDSessions 탭"conv-abc-123"
o.user사용자 식별자Users 탭"user-456"
o.q사용자 질의 텍스트대화 리플레이"What's the weather?"
o.r에이전트 응답 텍스트대화 리플레이"It's 72°F and sunny."

모범 사례

  • 일관성 있고 고유한 세션 ID를 사용하세요(UUID 권장).
  • 가시성 확보를 위해 o.sess와 o.user를 항상 함께 설정하세요.
  • 재연결 시에도 세션 ID가 유지되도록, 애플리케이션의 영구 대화 ID를 사용하세요.
  • 사용자 ID는 애플리케이션의 사용자 모델(email, UUID 등)과 일치시키세요.
  • A/B 테스트에서는 세션 ID에 prefix를 붙여 필터링을 쉽게 하세요. 예: exp-v2-{uuid}
