설치 가이드

로컬 개발, 프로덕션, 폐쇄망 등 다양한 환경에 EasyObs 서버를 설치하는 방법을 안내합니다.

시스템 요구사항

구성 요소최소권장
Python3.113.12
Node.js20.x20.x LTS
RAM512 MB2 GB+
Disk1 GB10 GB+ (트레이스 양에 따라 가변)
OSWindows 10+, macOS 12+, Ubuntu 22.04+

로컬 개발 환경 설정

1. 저장소 클론

git clone <repository-url>
cd easyobs_ai_observability

2. 환경 파일 생성

cp .env.sample .env

3. Python 가상 환경 & 의존성 설치

# uv 사용 (권장)
uv venv
uv pip install -e ".[dev,judges,pricing]"

# 또는 pip 사용
python -m venv .venv
source .venv/bin/activate        # Linux/macOS
.\.venv\Scripts\Activate.ps1     # Windows
pip install -e ".[dev,judges,pricing]"

4. 웹 콘솔 의존성 설치

cd apps/web
npm install
cd ../..

5. 개발 서버 시작

# 통합 스크립트 (API + Web 동시 실행)
./scripts/run-dev.sh             # Linux/macOS
.\scripts\run-dev.ps1            # Windows

프로덕션 배포

PostgreSQL 백엔드

프로덕션에서는 SQLite 대신 PostgreSQL을 사용하세요.

# .env
EASYOBS_DATABASE_URL=postgresql+asyncpg://user:password@host:5432/easyobs
⚠️ 주의: SQLite는 개발/데모 용도로만 사용하세요. 동시 접근이 발생하는 프로덕션 워크로드에서는 반드시 PostgreSQL을 사용해야 합니다.

API 서버 (프로덕션)

# 직접 실행
uvicorn easyobs.http_app:app --host 0.0.0.0 --port 8787 --workers 4

# 또는 환경 변수로 실행
EASYOBS_API_HOST=0.0.0.0 EASYOBS_API_PORT=8787 easyobs serve

웹 콘솔 빌드 (프로덕션)

cd apps/web
npm run build
npm start

폐쇄망(에어갭) / 오프라인 설치

PyPI에 접근할 수 없는 환경을 위한 설치 방법입니다.

1. 휠(wheel) 빌드 (인터넷이 가능한 머신에서)

.\scripts\build-agent-sdk.ps1 -IncludeDeps

2. 출력 구조

  • dist/agent/
    • easyobs_agent-0.1.0-py3-none-any.whl SDK 본체 (~15 KB)
    • deps/ 의존성 패키지
      • opentelemetry_api-*.whl
      • opentelemetry_sdk-*.whl
      • opentelemetry_exporter_otlp_proto_http-*.whl
      • ... (전이 의존성)

3. 대상 환경에 설치

pip install --no-index --find-links ./deps ./easyobs_agent-0.1.0-py3-none-any.whl

선택 의존성

Extra명령어설명
agentpip install -e ".[agent]"OpenTelemetry SDK, OTLP exporter 포함
langchainpip install -e ".[langchain]"agent + langchain-core 포함
judgespip install -e ".[judges]"OpenAI, Anthropic, Google GenAI, Bedrock 클라이언트
pricingpip install -e ".[pricing]"tokencost (LLM 가격 테이블)
devpip install -e ".[dev]"pytest, pytest-asyncio, ruff 등 개발 도구
