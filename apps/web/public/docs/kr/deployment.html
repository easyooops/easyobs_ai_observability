배포 가이드

PostgreSQL, 리버스 프록시, 영속 스토리지를 활용해 EasyObs를 운영 환경에 배포합니다.

아키텍처 (운영)

UI/UX 관점: 단일 공개 entrypoint가 브라우저 트래픽을 Web Console로, API 경로는 backend로 라우팅합니다. API는 DuckDB 쿼리 엔진을 통해 S3/클라우드 오브젝트 스토리지의 Parquet 파일을 직접 스캔하여 ClickHouse급 분석 성능을 제공합니다.

환경 변수 (운영)

# Database
EASYOBS_DATABASE_URL=postgresql+asyncpg://user:password@db-host:5432/easyobs

# Storage & Query Engine
EASYOBS_STORAGE_FORMAT=parquet
EASYOBS_QUERY_ENGINE=duckdb

# Server
EASYOBS_API_HOST=0.0.0.0
EASYOBS_API_PORT=8787
EASYOBS_CORS_ORIGINS=https://easyobs.yourcompany.com

# Security
EASYOBS_JWT_SECRET=your-strong-random-secret-here

# Logging
EASYOBS_LOG_FORMAT=json
EASYOBS_LOG_LEVEL=INFO

# Disable demo data
EASYOBS_SEED_MOCK_DATA=false

리버스 프록시 (Nginx 예시)

server {
    listen 443 ssl;
    server_name easyobs.yourcompany.com;

    # API
    location /v1/ {
        proxy_pass http://api:8787;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /otlp/ {
        proxy_pass http://api:8787;
        proxy_set_header Host $host;
    }

    location /healthz {
        proxy_pass http://api:8787;
    }

    # Web Console
    location / {
        proxy_pass http://web:3000;
        proxy_set_header Host $host;
    }
}

구성요소저장소권장

하이브리드 스토리지 아키텍처 (v0.2)

EasyObs v0.2부터 Hybrid 모드를 지원합니다. 로컬 Parquet Blob Store와 S3 아카이브를 동시에 사용하여 실시간 쿼리 성능과 장기 데이터 보관을 모두 충족합니다.
  • Ingest (수집): 모든 트레이스가 로컬 Parquet + S3에 동시 기록됩니다 (dual-write).
  • 쿼리 라우팅:
    • 프리셋 기간 (1h, 6h, 24h, 7d) → 로컬 DuckDB 엔진 (밀리초 응답)
    • Custom 기간 (7일 초과) → S3 DuckDB httpfs 엔진 (초 단위 응답)
  • 운영: 로컬 스토어는 EASYOBS_BLOB_HOT_RETENTION_DAYS(기본 7일)로 관리. 주기적으로 오래된 파티션을 정리할 수 있습니다.

스케일링

  • API Server — Uvicorn worker를 여러 개 실행: --workers 4 또는 프로세스 매니저 사용.
  • Web Console — 무상태이며 로드밸런서 뒤에서 수평 확장 가능.
  • Database — 커넥션 풀링 권장 (PgBouncer 또는 매니지드 풀).

보안 체크리스트

  • 강력한 EASYOBS_JWT_SECRET을 설정합니다 (openssl rand -hex 32).
  • HTTPS를 사용합니다 (프록시에서 TLS 종단).
  • EASYOBS_CORS_ORIGINS를 자사 도메인으로 제한합니다.
  • EASYOBS_LOG_REQUEST_BODY를 비활성화합니다 (PII 위험).
  • ingest 토큰을 주기적으로 회전합니다.
  • 데이터베이스를 네트워크적으로 격리합니다.
