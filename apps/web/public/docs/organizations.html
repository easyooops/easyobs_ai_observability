Organizations — Detailed Manual

Organizations — Full Feature Manual

EasyObs uses a multi-tenant organization model to isolate data, manage access control, and scope resources. Each organization is a complete, self-contained environment.

Hierarchy

UI perspective: users switch org context at the top, then work inside org-scoped modules. Service tokens are nested under Services.

Ingest Tokens

Each service can have multiple ingest tokens for key rotation:

  • Tokens are used as Authorization: Bearer <token> header in OTLP requests
  • Also used in SDK init(token="...")
  • Create multiple tokens for zero-downtime rotation
  • Revoke compromised tokens without disrupting other services

Token Lifecycle

  1. Create: PO clicks "Generate Token" → new token displayed once (copy immediately!)
  2. Active: Token is valid for authentication
  3. Revoke: Token is deactivated → SDK calls using it get 401 errors
Security: Tokens are only shown once at creation time. Store them securely (e.g., secrets manager, environment variable). They cannot be retrieved after the creation dialog is closed.

First User Bootstrap

When EasyObs starts with an empty database:

  1. First user to register automatically becomes Super Admin
  2. SA creates the first Organization
  3. SA invites other users and assigns roles
  4. Alternatively, if EASYOBS_AUTO_ORG is set, a default organization is auto-created

Organization Management

Creating an Organization

  1. Log in as Super Admin
  2. Navigate to Setup → Organizations
  3. Click "Create Organization"
  4. Enter name and optional description
  5. The creating user becomes the PO of the new org

Switching Organizations

  • The workspace header shows the current organization name
  • Click the org name → dropdown shows all orgs the user belongs to
  • Select a different org to switch context
  • All data (traces, runs, profiles) reloads for the selected org

Inviting Members

  1. Navigate to Setup → Members
  2. Click "Invite Member"
  3. Enter email address
  4. Select role: PO or DV
  5. Click Send Invite
  6. Invited user receives an email (or can register and be auto-added if matching domain)

Removing Members

  • PO/SA can remove members from the Members table
  • Removed users lose access immediately
  • Their historical labels and annotations remain

Data Isolation

  • Traces are scoped to organizations — members of Org A cannot see Org B's data
  • Evaluation profiles, judges, golden sets are all org-scoped
  • Ingest tokens are org-scoped — a token can only write to its own org
  • Super Admin can access all orgs for administrative purposes
