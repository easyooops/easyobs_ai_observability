Metrics & Dashboard ??Full Feature Manual

The Overview Dashboard is the landing page of the EasyObs workspace. It provides at-a-glance KPIs, trend charts, and quality summary for all ingested traces within the selected time window.

Trend Indicator Logic

  • Compares current window vs. equivalent previous period (e.g., last 1h vs. the 1h before that)
  • Green arrow ??for improvement (lower error rate, lower cost)
  • Red arrow ??for degradation (higher error rate, higher latency)
  • Gray dash ??for insufficient data

Hover over any point to see the exact value and timestamp in a tooltip.

Donut Chart (Row 3)

Shows the distribution of trace statuses:

  • OK (green segment) ??Successful traces
  • ERROR (red segment) ??Failed traces
  • UNSET (gray segment) ??Status not explicitly set

Center text shows the dominant status percentage.

Percentile Bars

Horizontal bars showing latency percentile distribution:

  • P50 ??Median latency (50th percentile)
  • P75 ??75th percentile
  • P90 ??90th percentile
  • P95 ??95th percentile (key SLA metric)
  • P99 ??99th percentile (tail latency)

Stacked Bars (Row 3)

Stacked horizontal bars breaking down usage by dimension:

  • By Service: Token/cost distribution across services
  • By Model: Which LLM models consume the most tokens
  • By Span Kind: agent / llm / retrieve / tool usage breakdown

Quality Panel (Row 4)

Summary card pulling from the Quality module:

  • Latest Pass Rate: From the most recent completed evaluation run
  • Trend: Pass rate over last 5 runs (mini sparkline)
  • Open Improvements: Count of improvement packs in "open" state
  • Link: "View Details ?? navigates to Quality > Runs

Pinned Alarms (Row 5)

Active alarms (FIRING state) are pinned to the dashboard:

  • Shows alarm name, current value vs. threshold, and duration
  • Red border for critical, yellow for warning
  • Click to navigate to Alarms page for details
  • Disappears automatically when alarm returns to OK state

Live Mode

  • Toggle "Live" in the time window control to enable auto-refresh
  • Dashboard data refreshes every 5 seconds
  • Time window rolls forward (always shows "latest N" period)
  • Useful for monitoring during deployments or load tests

Data Freshness

  • KPI data is computed server-side with staleTime: 30s
  • Non-live mode requires manual refresh or page reload for fresh data
  • A subtle "last updated X seconds ago" indicator shows data age
