Quality Evaluation

Quality Evaluation System

A comprehensive 3-Way evaluation framework combining Rule-based, LLM-as-a-Judge, and Human labeling approaches in a single unified platform.

📸 Image: Quality module overview page with KPI cards and recent runs

3-Way Evaluation

MethodSpeedCostBest For
Rule-basedInstant (on ingest)FreeFormat checks, length limits, keyword detection, regex patterns
LLM-as-a-JudgeSecondsPer-tokenSemantic quality, coherence, relevance, safety
Human LabelingHoursLaborGround truth, edge cases, subjective quality

Key Differentiators

  • Multi-Judge Consensus — Assign multiple LLMs to the same evaluation item. Choose from 4 consensus policies: single, majority, unanimous, weighted.
  • Cost Governance — Per-run, per-subject, and monthly budget caps with block, downgrade, or notify policies.
  • Auto-Rule on Ingest — Rule evaluations fire automatically when traces arrive (fire-and-forget, never blocks ingest).
  • Improvement Packs — Actionable suggestions derived from low-scoring evaluations, mapped to 52 metrics × 58 categories.
  • Golden Set Regression — Automated regression testing against curated ground-truth datasets.

Architecture

Trace Ingest
  └─ Auto-Rule Evaluation (fire-and-forget)
       └─ Rule DSL Engine (17 built-in rules)

Manual/Scheduled Trigger
  └─ LLM Judge Runner
       ├─ Multi-provider (OpenAI, Anthropic, Gemini, Bedrock)
       ├─ Consensus Engine (4 policies)
       └─ Cost Guard (budget enforcement)

Results → Runs → Improvements → Applied/Rejected

Module Components

Profiles

Define what to evaluate, which evaluators to use, and how to combine results.

Runs

Execute evaluations against traces and view pass/warn/fail results.

LLM Judges

Configure multi-provider LLM judge models with weights and temperature.

Golden Sets

Curate ground-truth datasets for regression testing and benchmarking.

Improvements

Actionable suggestions to improve agent quality based on evaluation results.

Enable/Disable

# .env
EASYOBS_EVAL_ENABLED=true               # Master switch
EASYOBS_EVAL_AUTO_RULE_ON_INGEST=true   # Auto-evaluate on trace arrival

When disabled, the Quality menu shows an informational page and no evaluation endpoints are registered.
