Deployment

Deployment Guide

Deploy EasyObs to production environments with PostgreSQL, reverse proxy, and persistent storage.

Architecture (Production)

+------------------------------------------+
|         Load Balancer                     |
|     (Nginx / ALB / Traefik)              |
+----------+---------------+---------------+
           |               |
  +--------+--------+  +---+----------+
  | API Server      |  | Web Console  |
  | (Uvicorn)       |  | (Next.js)    |
  | Port 8787       |  | Port 3000    |
  +--------+--------+  +--------------+
           |
  +--------+--------+  +--------------+
  | PostgreSQL      |  | Blob Store   |
  | (Catalog)       |  | (S3/Local)   |
  +-----------------+  +--------------+

Environment Variables (Production)

# Database
EASYOBS_DATABASE_URL=postgresql+asyncpg://user:password@db-host:5432/easyobs

# Server
EASYOBS_API_HOST=0.0.0.0
EASYOBS_API_PORT=8787
EASYOBS_CORS_ORIGINS=https://easyobs.yourcompany.com

# Security
EASYOBS_JWT_SECRET=your-strong-random-secret-here

# Logging
EASYOBS_LOG_FORMAT=json
EASYOBS_LOG_LEVEL=INFO

# Disable demo data
EASYOBS_SEED_MOCK_DATA=false

Reverse Proxy (Nginx Example)

server {
    listen 443 ssl;
    server_name easyobs.yourcompany.com;

    # API
    location /v1/ {
        proxy_pass http://api:8787;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /otlp/ {
        proxy_pass http://api:8787;
        proxy_set_header Host $host;
    }

    location /healthz {
        proxy_pass http://api:8787;
    }

    # Web Console
    location / {
        proxy_pass http://web:3000;
        proxy_set_header Host $host;
    }
}

Data Persistence

ComponentStorageRecommendation
Catalog (metadata)PostgreSQLManaged RDS/CloudSQL with backups
Trace blobsLocal FS / S3 / Azure Blob / GCSObject storage for scalability
JWT secretEnv var or secrets managerNever store in .env in production

Scaling

  • API Server — Run multiple Uvicorn workers: --workers 4 or use a process manager
  • Web Console — Stateless, horizontally scalable behind a load balancer
  • Database — Connection pooling recommended (PgBouncer or managed pool)

Health Checks

# API health
GET /healthz → {"status": "ok"}

# Use for: load balancer health checks, Kubernetes readiness/liveness probes

Security Checklist

  • Set strong EASYOBS_JWT_SECRET (openssl rand -hex 32)
  • Use HTTPS (TLS termination at proxy)
  • Restrict EASYOBS_CORS_ORIGINS to your domain
  • Disable EASYOBS_LOG_REQUEST_BODY (PII risk)
  • Rotate ingest tokens periodically
  • Network-isolate the database
