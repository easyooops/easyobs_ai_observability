# EasyObs on EC2 (Terraform)

Run from the **repository root**. Two topologies:

| Topology | Purpose | Terraform dir |
|----------|---------|---------------|
| **single** | One EC2: API + Web + Postgres (compose) | `setup/ec2/terraform/single/` |
| **cluster** | ALB + API leader EC2 + API worker ASG + Web EC2 + RDS Postgres + EFS (blob) | `setup/ec2/terraform/cluster/` |

**Worker scaling (OSS):** No separate queue workers (Celery/Dramatiq). Horizontal scale = run more API containers with the same image; ingest and auto-rule eval share the load. **Alarms:** only one instance should have `EASYOBS_ALARM_ENABLED=true` (duplicate alerts otherwise). In `cluster/`, Terraform user_data sets alarm on the leader only.

PowerShell: use the `.ps1` scripts instead of `.sh` where both exist.

---

## Prerequisites

- [Terraform](https://developer.hashicorp.com/terraform/install) ≥ 1.5
- [AWS CLI](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html) + credentials
- Repo checkout: Terraform zips the EasyObs tree to S3; EC2 downloads and `docker build`s on boot. **No ECR push required** for EasyObs images.

EC2 must pull base images (`python:3.12-slim`, `node:20-alpine`, `postgres:16`, …) from the internet. For air-gapped deploy, use `setup/offline/` (see §4).

## 1. Single node

### 1-1. Variables

```powershell
Copy-Item .\setup\ec2\terraform\single\terraform.tfvars.example `
          .\setup\ec2\terraform\single\terraform.tfvars
```

```bash
cp setup/ec2/terraform/single/terraform.tfvars.example \
   setup/ec2/terraform/single/terraform.tfvars
```

Key keys: `aws_region`, `instance_type`, `allow_easyobs_cidr`, `enable_data_volume`, `easyobs_api_image_tag`, `easyobs_web_image_tag`, `seed_mock_data`. `terraform.tfvars` is gitignored. Postgres password and JWT secret are generated by Terraform.

### 1-2. Apply

**Windows (PowerShell)** — from repo root:

```powershell
.\setup\ec2\terraform\single\tf-apply.ps1
```

**macOS / Linux / Git Bash:**

```bash
bash ./setup/ec2/terraform/single/tf-apply.sh
```

Flow: `terraform init` → `validate` → `plan -out tfplan`; confirm with `y`, then `terraform apply tfplan`.

### 1-3. Outputs

```powershell
cd .\setup\ec2\terraform\single
terraform output
terraform output -raw postgres_password
terraform output -raw jwt_secret
```

| Output | Meaning |
|--------|---------|
| `easyobs_api_url` | API base (`/docs`, `/healthz`, `/v1/...`) |
| `easyobs_web_url` | Next.js console |
| `public_ip` | EIP |
| `stage_bucket` | S3 staging bucket (debug) |

Bootstrap: `/var/log/easyobs-bootstrap.log`, SSM. ~5–10 min until `easyobs-api` is healthy.

First signup → super admin (`administrator` org). `seed_mock_data=true` seeds demo traces only if the catalog has no traces.

### 1-4. Destroy

```powershell
.\setup\ec2\terraform\single\tf-delete.ps1
```

```bash
bash ./setup/ec2/terraform/single/tf-delete.sh
```

## 2. Cluster

ALB + leader EC2 + worker ASG + Web EC2 + RDS + EFS. Scale workers via `api_worker_desired_capacity` or ASG.

```
                                  ┌─ Web EC2 (Next.js, 3000)
   Internet → ALB :80 ─┬─ /  ─────┘
                        │
                        └─ /v1/* /otlp/* /healthz /docs /openapi.json
                                 → API target group
                                      ├─ Leader EC2 (alarm=true)
                                      └─ Worker ASG (alarm=false, N)
                                           │
                                           └─ RDS Postgres + EFS (blob)
```

### 2-1. Variables

```bash
cp setup/ec2/terraform/cluster/terraform.tfvars.example \
   setup/ec2/terraform/cluster/terraform.tfvars
```

| Variable | Default | Notes |
|----------|---------|-------|
| `instance_type` | `t3.medium` | All EC2 (leader/worker/web) |
| `api_worker_min_size` | `1` | ASG min |
| `api_worker_desired_capacity` | `2` | Initial workers |
| `api_worker_max_size` | `6` | ASG max |
| `rds_instance_class` | `db.t3.medium` | |
| `rds_allocated_storage` | `50` GB | |
| `rds_multi_az` | `false` | Use `true` in prod |
| `rds_skip_final_snapshot` | `true` | Use `false` in prod |
| `allow_alb_cidr` | `0.0.0.0/0` | ALB ingress |

### 2-2. Apply

```powershell
.\setup\ec2\terraform\cluster\tf-apply.ps1
```

```bash
bash ./setup/ec2/terraform/cluster/tf-apply.sh
```

Typical time: **~15–20 min** (RDS, NAT, ASG, ALB).

### 2-3. Outputs

```bash
cd setup/ec2/terraform/cluster
terraform output
terraform output -raw rds_password
terraform output -raw jwt_secret
```

| Output | Meaning |
|--------|---------|
| `easyobs_url` | Entry (ALB DNS, HTTP) |
| `alb_dns_name` | ALB DNS (e.g. Route53 CNAME) |
| `api_leader_instance_id` | EC2 with alarms enabled |
| `api_worker_asg_name` | Worker ASG |
| `rds_endpoint` | Postgres |
| `efs_id` | Shared blob EFS |

### 2-4. Scale workers

```bash
aws autoscaling set-desired-capacity \
  --auto-scaling-group-name "$(terraform output -raw api_worker_asg_name)" \
  --desired-capacity 4
```

Or change `api_worker_desired_capacity` in `terraform.tfvars` and re-apply. New workers register to the ALB target group after user_data completes.

### 2-5. Destroy

```powershell
.\setup\ec2\terraform\cluster\tf-delete.ps1
```

```bash
bash ./setup/ec2/terraform/cluster/tf-delete.sh
```

If `rds_skip_final_snapshot=false`, destroy creates a final RDS snapshot.

## 4. Air-gapped deploy

EC2 Terraform assumes outbound internet for image pulls. Air-gapped: Docker image tar bundle + scripts in `setup/offline/`.

- Build: `setup/offline/build-bundle.sh` / `build-bundle.ps1`
- Guide: `setup/offline/README.md`

```bash
# 1) Build machine (online)
./setup/offline/build-bundle.sh --output ./dist/easyobs-bundle

# 2) Copy ./dist/easyobs-bundle to target, then:
cd /path/to/easyobs-bundle
./load-bundle.sh
./deploy-single.sh
# or
./deploy-cluster.sh single-host 4
./deploy-cluster.sh multi-host worker
```

Multi-host offline: shared NFS/EFS blob, external Postgres, internal LB; scripts take `EASYOBS_DATABASE_URL`, `EASYOBS_JWT_SECRET`, `EASYOBS_BLOB_HOST_DIR`, etc.

> **DuckDB + Parquet (v0.2+):** 모든 배포 모드에서 아래 두 환경 변수를 `.env`에 추가해야 한다.
>
> ```bash
> EASYOBS_STORAGE_FORMAT=parquet   # 권장. ndjson=레거시
> EASYOBS_QUERY_ENGINE=duckdb      # 권장. legacy=Python 루프
> ```
>
> S3/Azure/GCS blob 사용 시 `EASYOBS_BLOB_PROVIDER`, `EASYOBS_BLOB_BUCKET` 등을 `setup/compose/env.sample` 참조하여 설정.

## 5. LLM Judge — AWS Bedrock IAM 구성

EC2 인스턴스에서 AWS Bedrock Judge 를 사용하려면:

1. **IAM Instance Profile** 에 `bedrock:InvokeModel` 권한을 부여:
   ```json
   {
     "Effect": "Allow",
     "Action": ["bedrock:InvokeModel", "bedrock:InvokeModelWithResponseStream"],
     "Resource": "arn:aws:bedrock:*::foundation-model/*"
   }
   ```
2. Terraform `cluster/` 에서 EC2 role 에 위 정책을 추가하면,
   컨테이너 내에서 `AWS_PROFILE`/`AWS_ACCESS_KEY_ID` **없이도** boto3 가
   Instance Metadata Service(IMDS) 를 통해 자동 인증됨.
3. 로컬/docker 환경에서는 `.env` 에 `AWS_PROFILE=default` (named profile) 또는
   `AWS_ACCESS_KEY_ID` + `AWS_SECRET_ACCESS_KEY` 를 설정.
4. 리전은 Judge Model 등록 시 `connection.aws_region` 으로 지정 (기본 `us-east-1`).

> **기타 Provider (OpenAI/Anthropic/Google/Azure):**
> `.env` 에 `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `GOOGLE_API_KEY`, `AZURE_OPENAI_API_KEY` 등을 설정.
> 자세한 목록은 `setup/compose/env.sample` 의 "LLM Judge" 섹션 참조.

## 6. Destroy notes

- Prefer **Terraform destroy** for the stack; deleting only EC2 in the console can orphan VPC/EIP/ALB/ASG/EFS.
- **State:** destroy uses resources in the current dir’s `terraform.tfstate`. Lost state → manual cleanup in AWS.
- **S3 stage bucket:** `force_destroy=true` → objects removed on destroy.
- **EFS / RDS:** back up before destroy. RDS final snapshot when `rds_skip_final_snapshot=false`.
- Preview: `terraform plan -destroy` in the stack directory.
